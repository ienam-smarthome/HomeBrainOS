# HomeBrain Core v0.5 Device Engine

v0.5 introduces the first proper HomeBrain Core model:

- Maker API devices are normalised into consistent dictionaries.
- Device state is persisted to SQLite at `/data/homebrainos.sqlite3` inside the Home Assistant add-on.
- The dashboard and command engine read from the cache rather than parsing Maker API directly.
- Background refresh keeps the cache current.

## New API endpoints

- `GET /api/status`
- `GET /api/refresh`
- `GET /api/dashboard`
- `GET /api/devices?category=light&room=Hallway`
- `GET /api/rooms`
- `GET /api/device/{device_id}`
- `GET /api/ask?q=summary`

## Example questions
