# Hubitat MCP AI 0.4.41

A recognised semantic metric comparison must remain on the deterministic MCP executor. It must not silently fall through to the Cloud planner when one MCP response shape is rejected.

The executor now starts with the known-compatible compact `currentStates` capability read, treats detailed metadata as optional enrichment, catches each MCP request-shape failure independently, and reports structured evidence errors without asking Cloud to infer a numeric result.
